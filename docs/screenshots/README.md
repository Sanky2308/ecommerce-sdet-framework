# Portfolio Screenshots

Use this folder to store screenshots that make the framework easier for recruiters, clients, and hiring managers to understand.

## Recommended Screenshots

Add the following images here:

| File Name | What It Should Show |
|---|---|
| `allure-report.png` | Allure dashboard with passed/failed test summary |
| `github-actions.png` | Successful CI workflow run |
| `docker-run.png` | Tests running successfully inside Docker |
| `test-execution.png` | Local Maven/TestNG execution output |
| `trace-viewer.png` | Playwright trace viewer for failed UI debugging |

## How to Reference Screenshots in README

```markdown
![Allure Report](docs/screenshots/allure-report.png)
![GitHub Actions](docs/screenshots/github-actions.png)
![Docker Execution](docs/screenshots/docker-run.png)
```

## Screenshot Quality Rules

- Crop unnecessary browser or terminal noise.
- Keep private credentials, tokens, and emails hidden.
- Use clear names for each image.
- Prefer screenshots that prove execution, reporting, and CI/CD capability.

A repo without screenshots is technically fine, but visually it has the energy of an empty apartment listing with one blurry photo of a wall.
