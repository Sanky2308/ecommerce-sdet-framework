# Portfolio Polish Checklist

Use this checklist before sharing the repository with recruiters, hiring managers, or freelance clients.

## README

- [ ] Project purpose is clear in the first 5 lines
- [ ] Tech stack table is complete
- [ ] Framework architecture diagram is visible
- [ ] Folder structure is documented
- [ ] Local execution command is included
- [ ] Docker execution command is included
- [ ] Allure report command is included
- [ ] CI/CD section is included

## Framework Proof

- [ ] UI tests run successfully
- [ ] API tests run successfully
- [ ] E2E flow exists
- [ ] Screenshot capture works on failure
- [ ] Playwright trace works on failure
- [ ] TestNG suite runs from command line
- [ ] Parallel execution is stable
- [ ] Test data is externalized

## Reporting Proof

- [ ] Allure report screenshot added
- [ ] GitHub Actions screenshot added
- [ ] Docker execution screenshot added
- [ ] Local test execution screenshot added

## Professional Touches

- [ ] No hardcoded secrets
- [ ] No broken imports
- [ ] No commented-out junk code
- [ ] No personal test credentials committed
- [ ] Commit messages are clean
- [ ] Branch history is understandable

## Client/Recruiter Story

The repo should quickly communicate:

```text
I can design, build, run, debug, report, and maintain a professional automation framework.
```

That is the point. Not simply proving you can write one Playwright locator and then declare yourself an automation architect, which the internet already has in surplus.
